## 一、项目概述

2ASK光通信系统（嵌赛应用赛道）：
- **TX端**: NUCLEO-G474RE，DAC2输出50kHz载波，TIM2触发10kHz symbol rate
- **RX端**: STM32，ADC 200kHz DMA采样，LCD 128×128显示
- **通信距离**: 单向无线光通信
- **当前有效波特率**: 5kHz data bit（曼彻斯特编码后）

## 四、关键技术决策记录

1. **曼彻斯特编码**：解决长持续电平导致phase漂移的根本方案，每bit有跳变
2. **积分判决**：20 samples/symbol多数表决，对噪声和phase漂移鲁棒
3. **配对自动重同步**：收到非法对(00/11)时丢弃首symbol，重新配对
4. **GAP不编码**：16×0原始静默，用于break检测（曼彻斯特后无静默）
5. **FIFO翻倍**：曼彻斯特后每bit占2个FIFO位，FIFO大小需翻倍
6. **去掉所有手动skip**：曼彻斯特消除偏移，无需TYPE/LEN/CRC后的skip补偿
7. **LCD局部刷新**：Lcd_fill(x1,y1,x2,y2,color)即局部清屏，配合static缓存

## 六、调试要点（避坑指南）

### 6.1 通信问题排查

1. **NO SIG不恢复**：检查pattern_55/pattern_AA计数是否变化（ADC是否工作）
2. **FRM不自增**：检查CRC匹配（RX vs CL），示波器确认帧结构
3. **TYPE/LEN错误**：曼彻斯特配对是否错位（检查GAP后symbol对齐）
4. **CRC不稳定**：检查SIGNAL_HALF_SWING是否匹配实际信号摆幅

### 6.2 显示问题排查

1. **字符重叠**：Gui_DrawChar_Ascii需传bg参数填充背景
2. **全屏闪烁**：用Lcd_fill局部清屏，配合static缓存只更新变化字段
3. **颜色未定义**：LCDCYAN/MAGENTA需本地#define（Lcd_Driver.h未定义）
4. **白底标签不可见**：BIZ标签用GRAY2（深灰），不可用GRAY1（太浅）

### 6.3 硬件注意

1. **隔直电容**：已去除，DC耦合避免基线漂移
2. **TX UART**：ST-LINK虚拟串口用LPUART1，外部串口用USART1(PC4/PC5)
3. **按键**：PA0（模式切换），需Button_GPIO_Init()初始化
4. **ADC**：TIM3触发，200kHz，DMA buffer=20 samples
