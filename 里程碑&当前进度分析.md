# 光通信系统开发里程碑 & 当前进度分析

## 项目概况

**项目性质**: 2ASK光通信系统（紧急项目）  
**硬件平台**: STM32G474 (TX: 85MHz, RX: 150MHz)  
**通信方式**: LED发射 → 光电接收 → ADC采样  
**调试方式**: LCD显示（非USART）

## 二、核心算法实现

### TX端架构
```
UART命令 → ProcessCommand() → ASK_TX_SendFrame() → FIFO
                                                            ↓
                                                      帧编码
                                                      (0xAAE4 + TYPE + LEN + PAYLOAD + CRC)
                                                            ↓
                                                      TIM2中断(10kHz)
                                                            ↓
                                                      ASK_TX_FIFO_PopBit()
                                                            ↓
                                                      DMA切换(sinx[]/zero[])
                                                            ↓
                                                      DAC → LED发射
```

### RX端架构 - 两层处理
```
Layer 1: ADC → 阈值判决 → 边沿检测 → bit采样
Layer 2: bit流 → 帧解析状态机 → RxFrame_t

详细流程:
ADC 200kHz采样 → DMA中断(每20个样本)
    ↓
边沿检测(0→1/1→0) → 相位重置
    ↓
相位计数 → 在中点(phase=10)采样
    ↓
bit流 → 帧解析状态机
    ↓
FRAME_SEARCH → 检测0xAAE4帧头
    ↓
FRAME_RECV_TYPE → 接收TYPE字节
    ↓
FRAME_RECV_LEN → 接收LEN字节
    ↓
FRAME_RECV_PAYLOAD → 接收PAYLOAD
    ↓
FRAME_RECV_CRC → 接收CRC并校验
    ↓
有效帧 → frame_ready=1
```

## 四、验证步骤

### 1. TX端独立验证
```
串口发送: H
预期输出: FIFO: XXX bits, Signal: X

串口发送: D
预期输出: 显示帧编码示例

串口发送: RTest
预期输出: 
=== Frame Encoding Debug ===
Type:  0x01
Len:   4 bytes
Frame (hex): AA E4 01 04 54 65 73 74 XX XX
```

### 2. RX端独立验证（无TX信号）
LCD应显示:
```
ONES:   0%
LAST:  0x0000
BITS:  00000000
ADC:   XXXX     ← 观察ADC值是否合理
THR:   2006
NO SIGNAL
```

### 3. 双端联调验证
**步骤1**: TX发送空闲信号（默认自动发0x55）

**步骤2**: RX观察LCD
- ONES: 应稳定在50%左右
- BITS: 应显示01010101或10101010
- LAST: 应在0x5555和0xAAAA之间跳动
- PAT: 应显示"0x55 OK"或"0xAA OK"

**步骤3**: TX发送测试帧
```
串口发送: RHello
```

**步骤4**: RX接收验证
- 观察LCD的frame_count是否增加
- 通过`SignalProcessor_GetFrame()`读取帧内容
- 验证TYPE=0x01, LEN=5, PAYLOAD="Hello"

### 4. 阈值调整
如果ONES%不在40-60%范围:
1. 观察LCD的ADC值
2. 调整`ENVELOPE_THRESHOLD`使阈值在ADC高低值中间

## 六、关键技术点

### 1. 帧头设计
```
帧头: 0xAAE4
- 0xAA (10101010): 时钟恢复/AGC
- 0xE4 (11100100): 帧同步标记

关键优势: 0xE4 NEVER出现在空闲0x55流中 → 零误触发
```

### 2. 位同步算法
```
边沿检测: bit != last_bit → phase重置为0
相位计数: phase++
中点采样: phase == 10 (SAMPLES_PER_BIT/2)

优势: 
- 采样点始终在bit中心（最稳定）
- 自动适应时钟偏差
- 简单高效
```

### 3. CMSIS类型定义冲突
```
问题: 用户头文件先于CMSIS包含<stdint.h>
解决: 所有.c文件首先包含main.h

正确顺序:
main.h → stm32g4xx_hal.h → core_cm4.h → <stdint.h>
```

## 八、已知问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 编译错误: `uint16_t is not a type name` | CMSIS类型定义顺序冲突 | .c文件首先包含main.h |
| 编译错误: `__INLINE undefined` | Keil MDK不支持该宏 | 改为`inline`或分离声明/定义 |
| LCD字符重叠 | 局部刷新未清除完整 | 使用Lcd_fill清除区域 |
| ONES%全0或全1 | 滤波器窗口过大/采样率不匹配 | 调整TIM3采样率+去除滤波 |
| 误码率高 | 无位同步算法 | 实现边沿检测+相位重置 |
